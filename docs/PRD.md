# 堆方块大作战 — 产品需求文档

## 问题陈述

目前微信小游戏平台缺乏一款「操作简单但深度足够」的异步竞技社交游戏。现有堆叠类游戏多为纯单机，玩家缺乏与他人较劲、互动的动力，留存和传播受限。

## 解决方案

基于原型「堆方块」核心玩法（操控小车接住掉落方块、堆叠同色合并、维持平衡不倒），打造一款以**异步竞技排行榜**为核心、融合好友挑战、幽灵回放、群排行和每周赛季的高社交属性微信小游戏。

## 用户故事

1. 作为一名玩家，我希望能通过微信直接进入游戏，无需注册登录，立刻开始玩

8. 作为一名玩家，我希望能使用技能（缓降、磁力、同色化、倾斜复位）来逆转局势
9. 作为一名玩家，当我堆太高导致底座倾斜时，我希望能感受到紧张并努力稳住
10. 作为一名玩家，当堆叠倒塌时，我希望能有一次看广告复活的机会
11. 作为一名竞技玩家，我希望在天梯对战后，看到自己在微信好友中的排名
12. 作为一名竞技玩家，我希望看到好友排行榜，清楚地知道自己超过/落后于哪些好友
13. 作为一名竞技玩家，我希望打出高分后，能一键挑战微信好友，对方会收到我的挑战
14. 作为一名被挑战的玩家，我希望打开挑战后，游戏顶部始终显示发起者的分数，我打完就知道赢了还是输了
15. 作为一名玩家，我希望能观看好友的对战回放，学习操作技巧
16. 作为一名玩家，我希望在微信群里能看到群内成员的排行，跟群友互相较劲
17. 作为一名竞技玩家，我希望能参与每周赛季，每周日晚上 8 点结算排名
18. 作为一名赛季参与者，我希望根据赛季排名获得对应的称号/勋章（铜/银/金/钻石）
19. 作为一名收藏爱好者，我希望能用游戏金币购买不同的小车外观
20. 作为一名收藏爱好者，我希望能解锁不同风格的方块主题（水晶方块、像素方块等）
21. 作为一名付费意愿玩家，我希望能通过看激励广告获取更多金币
22. 作为游戏运营方，我需要在每 3 局游戏后展示一次插屏广告以获取收益
23. 作为一名玩家，我希望能看到自己的对战历史记录
24. 作为一名玩家，我希望能看到自己拥有的皮肤和已解锁内容

## 实现决策

### 技术栈

- **前端**：微信小游戏原生 Canvas API（纯 JavaScript）
- **后端**：微信云开发（CloudBase）
  - 排行榜、群排行：微信开放数据域（`wx.getFriendCloudStorage`）
  - 挑战记录、回放数据、赛季结算：云数据库 + 云函数
- **广告**：微信流量主广告组件（插屏广告 + 激励视频广告）

### 代码架构（从原型重构）

从原型 `weixinxiaochengxutest` 重构为 5 层模块化结构：

```
gameone/
├── game.js                     # 入口
├── game.json                   # 小游戏配置
├── project.config.json
├── engine/
│   ├── physics.js              # 重力、碰撞、倒塌、倾斜
│   ├── merger.js               # 同色方块合并（含特殊方块逻辑）
│   ├── spawner.js              # 方块生成（颜色、概率、随机种子）
│   └── car.js                  # 小车移动、宽度、碰撞检测
├── data/
│   ├── databus.js              # 游戏运行时状态
│   ├── storage.js              # 本地持久化
│   └── replay.js               # 回放记录 + 回放播放器
├── ui/
│   ├── menu.js                 # 主菜单
│   ├── hud.js                  # 游戏内 HUD
│   ├── leaderboard.js          # 排行榜
│   ├── challenge.js            # 好友挑战界面
│   └── shop.js                 # 皮肤商店
├── social/
│   ├── open-data.js            # 开放数据域通信
│   └── challenge.js            # 挑战流程
└── cloud/
    └── functions/              # 云函数
```

### 天梯模式参数（锁定）

| 参数 | 值 |
|------|-----|
| 颜色种类 | 6（红/蓝/绿/黄/紫/橙） |
| 特殊方块生成概率 | 4%（炸弹/磁铁/万能各约 1.33%） |
| 初始生成间隔 | 2.2 秒 |
| 最小生成间隔 | 0.7 秒 |
| 重力 | 900 |
| 车速 | 固定 |
| 车宽 | 固定（76px） |
| 生命数 | 5 |
| 技能 | 缓降 1 次、磁铁 2 次、同色化 1 次、复位 1 次 |
| 升级 | 天梯模式下所有升级无效 |
| 复活 | 天梯模式下禁用复活 |

### 回放系统

- 存储内容：触摸坐标序列 + 时间戳 + 随机种子
- 回放方式：用相同种子重新运行游戏逻辑，根据操作序列驱动
- 每局数据量：预计 30-80KB
- 存储位置：云开发集合 `replays`，保留最近 7 天

### 好友挑战流程

1. 玩家 A 打完天梯 → 点击「挑战好友」→ 调用 `wx.shareAppMessage` 生成分享卡片
2. 好友 B 点击卡片 → 进入游戏 → 进入挑战模式
3. 挑战模式：天梯规则 + 顶部固定显示 A 的分数标杆 +「超越目标」标记
4. B 打完 → 判定胜负 → 结果写入「我 vs A」双方可见 → 通知 A

### 赛季机制

- 周期：每周一 00:00 ~ 周日 19:59
- 结算：周日 20:00，云函数自动计算排名
- 称号：前 50% 青铜、前 20% 白银、前 5% 黄金、前 1% 钻石
- 历史：显示「本周排名」「上周称号」「历史最佳」

### 皮肤系统

- **小车皮肤**：金币购买 + 赛季奖励限定
- **方块主题**：金币购买（水晶/像素/果冻/金属等风格）
- 金币来源：游戏内每局获得 + 看激励广告额外获取
- 存储：本地持久化 + 云端同步

### 广告策略

- 插屏广告：每 3 局天梯模式结束后展示 1 次
- 激励视频广告：原模式复活、双倍金币
- 天梯模式：无广告干扰（保证竞技公平）

## 测试决策

### 测试标准

- 只测试公开接口的外部行为，不测试实现细节
- 测试应能经受内部重构而不失效
- 每个测试描述一个行为：「某条件下，输入 X，期望输出 Y」

### 待测试模块

| 模块 | 测试重点 |
|------|---------|
| `engine/physics` | 重力下落、小车碰撞检测、稳定性计算、倒塌触发条件、倾斜角度累计 |
| `engine/merger` | 同色方块的合并判定（横向/纵向）、特殊方块合并逻辑（炸弹引爆范围、磁铁吸附、万能色匹配） |
| `engine/spawner` | 随机种子确定性、颜色分布、特殊方块生成概率、生成间隔变化 |
| `data/replay` | 操作序列编解码、回放状态还原一致性、随机种子复现 |
